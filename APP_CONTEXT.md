# Big Win - Full Application Technical Dossier

## 1. High-Level Overview
**App Name:** Big Win (also referred to as Big WinGo)
**Category:** High-frequency color prediction betting platform.
**Target Platform:** Mobile-first Web (Optimized for 60FPS on low-end Android devices).
**Core Stack:** 
- **Frontend:** React 19, Vite 7, TypeScript.
- **Styling:** Tailwind CSS 4 (using @tailwindcss/vite).
- **State:** Zustand (Global game engine, timers, and UI state).
- **Animations:** Framer Motion (Optimized with GPU-accelerated CSS transitions).
- **Backend:** Supabase (PostgreSQL, Auth, Realtime, Edge Functions/RPCs).

## 2. The Game Engine (The Logic)
The app runs multiple concurrent game sessions (30s, 1m, 3m, 5m, 10m).

### A. The "Ruthless" House Logic
The system is mathematically guaranteed to never lose money in high-traffic scenarios.
1. **Simulation:** At the end of every round, the engine calculates the total payout exposure for all 10 possible numbers (0-9).
2. **Calculation:** It factors in bets on Colors (Green/Red/Violet), Sizes (Big/Small), and individual Numbers.
3. **Selection:** It selects the outcome(s) that result in the **absolute lowest total payout** from the house.

### B. Low Traffic Protection (Trust Builder)
Controlled via Admin Panel. If `low_traffic_mode` is ON and total bets are below `low_traffic_threshold` (e.g., ₹1500):
- The engine identifies all outcomes that have **at least one real user bet**.
- Among *only* those outcomes, it picks the one with the lowest payout.
- **Result:** In low traffic, users win occasionally to build trust, but the house still minimizes loss.

### C. Synchronization & Reliability
- **Tick Engine:** Uses `requestAnimationFrame` for a smooth heartbeat.
- **Idempotency:** Uses a `resolvingPeriods` re-entrancy guard to prevent double-resolving a round.
- **Authoritative DB Fetch:** If a client fails to insert a result (due to unique constraint), it enters an exponential backoff loop to fetch the result generated by the "winning" client.

## 3. Financial Architecture (The 3-Tier Wallet)
The app uses a segmented wallet system stored in the `profiles` table:
1. **`deposit_balance`:** Funds added via UPI (Locked, must be used for betting).
2. **`bonus_balance`:** Marketing funds (Joining bonus, Loss bonus, Referrals).
3. **`winnings_balance`:** Profit from games (The only withdrawable balance).
4. **`balance`:** A calculated total of all three (Usable for betting).

### Deduction Hierarchy:
When a bet is placed, funds are deducted in this order:
**Bonus Balance -> Deposit Balance -> Winnings Balance.**

---

## 4. Automated Bonus Systems
- **Joining Bonus:** Configurable fixed amount (₹80) + optional % of first deposit. Unlocked automatically upon the first admin-approved deposit.
- **Loss Bonus:** Automated "Cashback." If a user has a net loss in a round, the system immediately grants XX% (configurable, default 10%) of that loss back into the `bonus_balance`.
- **Referral Bonus:** Tiered percentage based on referred user deposits and game losses.

---

## 5. Database Schema & Security
### Key Tables:
- `profiles`: User data, 3-tier balances, admin status, referral links.
- `bets`: Real-time bet tracking, payout status, and round association.
- `games`: Authoritative history of every round result.
- `transactions`: Immutable log of every rupee moved (metadata column stores round IDs).
- `app_settings`: Global toggles (Low Traffic Mode, Bonus %, UPI IDs).
- `support_tickets`: In-app WhatsApp-style chat system.

### Security:
- **RLS (Row Level Security):** Users can only read their own rows. Admins have bypass access.
- **RPCs (Remote Procedure Calls):** No balance logic exists on the frontend. All money movement happens inside secure PostgreSQL functions (`update_user_balance`, `admin_process_deposit`, etc.).

## 6. Admin Panel Capabilities
- **Real-time Analytics:** 7-day profit/loss charts and live user stats.
- **Force Result:** Ability to manually override the next round's outcome for any session.

- **User Management:** Manual balance adjustment and role assignment.
- **Global Settings:** Live toggles for Bonus %, Thresholds, and Support links.
